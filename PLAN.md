## 0. この文書の使い方

これから先の作業は、主に **Claude Code**（コードを実際に書いてくれるAIツール）を使い、`dev-orchestration-template`（ひな形）の「計画→実装→三重監査→リリース」ワークフローで進めます。このPLAN.mdをClaude Codeに読み込ませて、「このプランのタスク1-1から始めてください」のように指示すれば、そのまま作業に着手できる形にしてあります。

タスクには次の3つの印をつけています。

- 🟦 **決める作業**：英理究さんが決めること（コードは書かない）
- 🟩 **作る作業**：Claude Codeにコードを書いてもらう作業
- 🟨 **確認する作業**：できあがったものを、英理究さんが実際に触って確認する作業

## 1. 全体の進み方（4つの大きな山）

仕様書9章の Phase 1〜4 を、実際の作業の山として並べ直すと、次のようになります。

```
山1：土台づくり          → データの入れ物（Firestore）とログインの仕組みを作る
山2：算数・数学ドリル本体  → テンプレート方式の出題・採点・ヒント機能を作る（アプリの主役）
山3：仕上げ              → 保護者ダッシュボード、実機での動作確認
山4：将来の拡張          → AI文章問題、英語教科（このPLANでは詳細化しない。今後別途プランを作る）
```

このPLAN.mdでは、**山1〜山3（＝仕様書のPhase 1に相当する部分）を、実際に手を動かせる粒度まで分解**します。山4（Phase 2以降）は仕様書ですでに設計済みのため、山1〜3が完成した後にあらためてプランを作ります。

## 3. 直近の進め方（今すぐ着手する順番）

上の表の「🔴 最優先」から順に進めます。実際の作業順は次の通りです。

1. **タスク1-1・1-2を先に決める**（英理究さんの判断が必要。ここが決まらないと、テンプレートの中身が決められません）
2. タスク1-3・1-4で、開発の土台を用意する（Claude Codeと一緒に作業）
3. タスク2-1〜2-3で、アプリの骨組み（ログイン→単元選択→出題）を作る。この時点で、テンプレートは1つだけ（たとえば「かけ算」）でよいので、**動くものを早く見る**ことを優先する
4. タスク2-4〜2-6で、決めた単元の分だけテンプレートと図形描画を増やしていく
5. タスク2-7〜3-4で、記録・保護者画面・見た目を仕上げる

> ポイント：**2の「骨組み」が動いた時点で一度、実際にブラウザで触ってみることを強くおすすめします。** 土台の使い勝手に問題があれば、単元を増やす前に直したほうが、あとの手戻りが少なくなります。
